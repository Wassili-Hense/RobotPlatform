Arduino / ESP32-D0WD-V3

1. HMI-модуль на ESP32
- Использовать hmi.h / hmi.cpp как клиент HMI.
- hmi_tick() должен вызываться периодически каждые 5 мс.
- После hmi_tick() должна отправляться не более чем одна команда HMI.
- Любая LCD-команда может отправляться только если hmi_get(HMI_DATA_STAT_LCD_BUSY) == 0.
- hmi_init() принимает callback для логирования.
- Если DEBUG_HMI определён, hmi.cpp должен отправлять debug/error сообщения через callback.
- При ошибках callback вызывается с текстом формата "<имя функции> - <имя ошибки>" и emergency=true.
- После ParsePacket() в hmi_tick(), если есть изменения данных, должен выводиться debug-статус в формате:
  RX %1X %5u %5u %s
- Перед возвратом HMI_CMD_OK из SendCommand() должен логироваться текст "TX " и отправленные байты.

2. Валидация аргументов HMI-команд
- Для LCD text разрешены только символы charset[32..126].
- font 7x10
- Для LCD text и marker координаты должны находиться внутри доступной области экрана x=5..154, y=5..79.
- Для команд с неверными аргументами должен возвращаться HMI_CMD_ERR_INVALID_ARG, чтобы такие команды не повторялись бесконечно.
- Для временно недоступных команд (например LCD busy / I2C TX) должен сохраняться признак возможности повторной попытки.

3. Организация очереди команд на ESP32
- Описание структуры HMI-команды, unions config/state, типы команд, send-функции, helper-макросы и HmiqServiceOne() вынести в hmi_helpers.h.
- В .ino работать напрямую с отдельными переменными команд, например:
  s_cmdUsbIndicator
  s_cmdRfIndicator
  s_cmdProgress0
  s_cmdProgress1
  s_cmdProgress2
  s_cmdBeep
- После объявления этих переменных собрать массив s_hmiSys из указателей на них.
- В .ino обновлять state/hasData именно у s_cmd* переменных напрямую, без поискового helper'а вроде HmiqFind.

4. Serial / USB поведение
- Если изменился HMI_DATA_STAT_USB_CONN:
  - при 1: Serial.begin(115200), затем очередь на LCD_Indicator(1, 1)
  - при 0: Serial.end(), затем очередь на LCD_Indicator(1, 0)
- Если log callback приходит с emergency=true, Serial должен включаться принудительно перед выводом сообщения.

5. Приём команд из Serial
- Реализовать текстовый приём команд в .ino:
  ANN  -> DrawProgressBar(0, NN)
  BNN  -> DrawProgressBar(1, NN)
  CNN  -> DrawProgressBar(2, NN)
  DN   -> LCD_Indicator(0, N)
  THz,Ms -> Tone(1000000/Hz, Ms)
- Команды должны парситься построчно по CR/LF.
- Некорректные команды должны игнорироваться без аварий.

5. Джойстик и маркер
Обычный режим

Рисовать положение джойстика как маркер:

LCD_DrawMarker(idx = 3, color = белый)
область: x=45..114, y=10..79
Старый маркер стирать чёрным
если LCD backlight state == off, стереть и не обновлять

Калибровка

Cal center

LCD_DrawMarker(idx = 5, color = зелёный)
старые маркеры не стирать
OK — сохранить
BACK — не сохранять
Маштабировать поле вывода. w=h=2*max(предыдужие x.cMax - x.CMin, y.cMax - y.cMin, 16), center - (x.cMax + x.CMin)/2, (y.cMax + y.cMin) / 2
Ширина dead zone не менее 30.

Cal edge

LCD_DrawMarker(idx = 3, color = синий)
старые маркеры не стирать
OK — сохранить
BACK — не сохранять


6. Калибровка осей
Для каждой оси хранить 4 значения:

[min, cMin, cMax, max]

Интерпретация:

min  → -1
cMin → 0
cMax → 0
max  → 1

Значения сохранять в Preferences.

7. Меню
Вход

Кнопка OK открывает меню

Пункты

Cal center
Cal edge

Управление в меню

UP / DOWN — выбор пункта
OK — вход / подтверждение
BACK — назад / выход
LUP / LDN — регулировка яркости

Текст калибровки

Текст режимов калибровки рисовать справа, вне зоны маркера, вертикально(через /n), Слева - cancel, справа - accept


8. Яркость и подсветка
Яркость

Хранить в 11 логарифмических шагах в диапазоне[1..127]
Сохранять в Preferences

Поведение

при входе в меню:

SendBacklightBrightness(s_backlightLevel)


при выходе из меню:

SendBacklightBrightness(max(s_backlightLevel / 2, 1))


при изменении яркости в меню:

обновлять только поле яркости, без полной перерисовки меню



Timeout подсветки

пока меню активно:

слать SendBacklightTimeout(15000) каждые 10 секунд



После старта

загрузить яркость и калибровку из Preferences
яркость отправлять только после первого успешно принятого STATUS
отправлять max(s_backlightLevel / 2, 1)


9. Выход из меню по не активности

Если 60 секунд нет активности — выйти из меню
Активность = только изменение состояния кнопок
Джойстик на таймер неактивности не влияет


10. Команды из Serial
Поддержать команды:

ANN → DrawProgressBar(0, value)
BNN → DrawProgressBar(1, value)
CNN → DrawProgressBar(2, value)
DN  → LCD_Indicator(0, value)
THz,Ms → Tone(1000000/Hz, Ms)
