//Arduino, ESP32-D0WD-V3 (revision v3.1), SDA = 21, SCL  = 22, addr = 20;
Экран(подключен к I2C slave): R5G6B5, charset[32 .. 126],Font 10x7, доступная область x0=5, y0=5, x1=155, y1=79. 

читает с I2C slave, период опроса 20 мс.
в Serial выводятся данные без заголовков разделённые " ":
- индекс из пришедшей комманды (%1x)
- статус (%d) 
- X и Y. [-1 .. 1] в формате (%.3f) 
- имя нажатой кнопки(без BUTTON_), если несколько, то с меньшим индексом 
печать по приходу обновления.

если LCD busy=1, команды LCD не шлются, код ждёт освобождения очереди. Статус LCD busy приходит из STATUS bit0.
если Usb connected=0{
  вызывается Serial.end()
  рисует в правом верхнем углу чёрный квадрат 5x5
}
если 1{
  Serial.begin(115200)
  рисует в правом верхнем углу синий квадрат 5x5
}

рисует положение джойстика как белый маркер FillCircle(r=3) в области x=10..79, y=10..79;
старый маркер стирает чёрным;

по OK открывает меню;
пункты меню:

Cal center — калибровка центра, до OK - сохранить изменения/BACK - не сохранять, маркер зелёный, старые не стираются;
Cal edge — калибровка края, до OK/BACK, маркер синий, старые не стираются;
Для каждой оси джойстика хранится 4 переменных: [min, cMin, cMax, max] с переводом [-1, 0 ,0 1] 

пока меню активно:

UP/DOWN — выбор пункта;
OK — вход/подтверждение;
BACK — назад/выход;
LUP/LDN — регулировка яркости;


настройки джойстика и яркости сохраняются в Preferences;
яркость после старта отправляется в STM32 после первого успешно принятого ответа;
команды из Serial:

ANN → DrawProgressBar(0, value)
BNN → DrawProgressBar(1, value)
CNN → DrawProgressBar(2, value)
THz,Ms → Tone(1000000/Hz, Ms)



// ===================================================================================================================================

LCD busy - очередь сообщений для LCD переполнена, ждать пока освободится.
Usb connected - 0 - Serial.end(), 1 - Serial.begin(115200) и общение по Serial разрешено.

По умолчанию, на экран выводится положение джойстика [70, 70](FillCircle(r=3, white), старый стирать
по нажатию OK вывод меню.
Пункты меню:
 - калибровка джойстика центр, до нажатия OK или BACK(маркер зелёный, не стирать)
 - калибровка джойстика край, до нажатия OK или BACK(маркер синий, не стирать) 
Пока меню активно, LUP и LDN регулируют яркость подсветки.
Джойстик не обновляет сам BacklightTimeout

Команды A, B, C с Serial отсылают DrawProgressBar.
На Serial отсылать X и Y джойстика в формате [-1 .. 1]
Команда Т  с параметрами герц и миллисекунд через запятую.

Настройки джойстика и яркости сохранять в постоянной памяти, яркость при старте отсылать на устройство после первого принятого ответа.
